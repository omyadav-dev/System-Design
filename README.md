Day 01: Client-Server Architecture & Proxies📌 IntroductionClient-Server Architecture is the fundamental building block of the Web. It is a distributed framework that divides tasks between service providers (Servers) and service requesters (Clients).1. The Core: Request & Response ModelWeb interaction follows a simple loop:The Client: A front-end application (web browser, mobile app) that initiates a request for data or an action.The Server: A powerful machine running 24/7, waiting to process requests, perform operations (CRUD), and return a response.🌐 Identity on the Web: IP & DNSIP Address: A unique identifier for every machine, acting like a phone number for the server.DNS (Domain Name System): A distributed database that maps human-friendly domain names to numerical IP addresses.Workflow: When you type a URL, the browser first asks the DNS for the IP; once retrieved, it establishes a direct connection to the actual server.🛡️ Intermediaries: Proxy vs. Reverse ProxyProxies act as middlemen to handle security, privacy, and speed.A. Forward Proxy (Protects the Client)Sits between a private network and the internet.Anonymity: The server only sees the proxy IP, hiding the client's identity.Content Filtering: Used by organizations to block specific websites.Caching: Stores frequently accessed pages to reduce latency (response time).B. Reverse Proxy (Protects the Server)Sits in front of backend servers to manage incoming traffic.Security: Hides the actual server IP to prevent DDoS attacks.Load Balancing: Distributes requests across multiple servers (A, B, C) so no single machine is overloaded.SSL Termination: Handles encryption/decryption so backend servers can focus on processing data.💡 ML Engineering Interview ScenariosWhen applying these concepts to Machine Learning deployments:If a Model is Slow: Check Latency. Is the delay in the network (DNS/Proxy) or the model inference itself? Add a forward proxy with caching for frequent inputs.If Server Crashes During Peak Time: Use a Reverse Proxy as a Load Balancer to distribute traffic so no single server is overloaded.Protecting Model Server IP: Use a Reverse Proxy so the client only interacts with the Proxy IP, keeping internal backend infrastructure hidden and secure.📊 Key Comparison FeatureForward ProxyReverse ProxyActing forClientServerLocationInternal / Private NetworkEdge of the Server NetworkIP HiddenClient IP is hiddenServer IP is hiddenKey FunctionPrivacy / Bypassing RestrictionsLoad Balancing / Security / Speed
